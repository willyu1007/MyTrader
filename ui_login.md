# 登录/创建账号界面（UI 规范）

本文件用于记录 MyTrader 的“登录 / 创建账号”界面外观与交互规范，作为实现与验收的共同依据。

## 目标
- 复刻“专业金融终端”风格：深色为主、青色强调、边框与微光阴影、输入框左侧图标。
- 结构清晰：上半部分登录、下半部分创建账号，中间用分隔线与“或”提示。
- 全中文：界面文案不出现英文。
- 跟随系统主题：浅色/深色自动切换（`prefers-color-scheme`）。
- 本地优先：不依赖外部 CDN 字体/图标；使用项目内 CSS token（`apps/frontend/src/theme.css`）。

## 页面结构
### 顶部栏
- 左侧：品牌方块（“M”） + 标题“MyTrader”。
- 右侧：等宽 badge “MVP 基础”。

### 主体
- 居中大卡片（宽度约 860px 上限）。
- 深色模式下主区域可有两团模糊光晕（青色 + 蓝色）增强氛围。

### 分区 1：登录
- 标题：左侧青色竖条 + “登录”。
- 表单：
  - 账号：支持输入 + 下拉建议（datalist），输入框左侧图标；右侧可有下拉箭头装饰。
  - 密码：密码输入框左侧锁图标。
  - 登录按钮：右对齐，主按钮（填充青色/青色渐变），带轻微发光阴影。

### 分隔线
- 一条横线，中间文案“或”。

### 分区 2：创建账号
- 标题：左侧青色竖条 + “创建账号”。
- 表单：
  - 名称：文本输入框左侧图标。
  - 密码：密码输入框左侧钥匙图标。
  - 确认密码：密码输入框左侧对勾图标；当两次密码不一致时在下方显示红色提示。
  - 数据目录：
    - 左侧输入框（可粘贴路径），左侧文件夹图标。
    - 右侧“选择…”按钮：次按钮（灰色/中性），点击打开系统目录选择器。
    - 下方说明：该目录用于存放账号数据；创建后在该目录下生成独立账号文件夹及 `business.sqlite`、`analysis.duckdb`。
  - 创建按钮：描边主色按钮（hover 时可转为填充/更强调）。

### 告警条
- 信息提示（info）：当未检测到桌面端后端（preload API）时显示，说明此时登录/创建/目录选择不可用。
- 错误提示（error）：发生系统错误时显示红色告警条，含图标、标题“系统错误”与错误信息。

### 页脚
- 居中小字版权信息（可简短）。

## 交互与状态
- 账号列表：来自桌面端 IPC；账号输入支持自由输入与建议。
- 登录：需要账号 + 密码；失败时在错误告警条展示信息。
- 创建：需要名称 + 密码 + 确认密码一致；失败时在错误告警条展示信息。
- 数据目录选择：
  - 点击“选择…”后显示“打开中…”状态；若弹窗未出现，提示可能被遮挡。
  - 取消选择不改变当前输入。
